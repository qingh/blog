1.安装eslint

全局安装或是安装在项目里面。

npm i eslint -g  或 npm i eslint --save-dev

区别是，全局安装，可以在任何地方使用eslint命令。项目里面安装要使用在package.json文件里面配置，例

"scripts": {
    "eslint": "eslint ./public/js/*.js"
}

然后使用 npm run eslint 执行检查

2.初始化一个eslint规则文件 .eslintrc.js

eslint --init

此命令会提示你，需要什么样的规则去进行检查，根据自己偏好进行选择即可。此时我用的是推荐设置

"extends": "eslint:recommended",

3.下面就可以去检查你的代码质量了

eg : eslint app.js

4.初次检查了整个项目，报了好几千个error，可能我设置的规则比较严格，但都是小问题。下面说一下几个常见问题及解决方案

a. '$' is not defined no-undef

因为我的项目依赖了jquery，所以会报这个错误。解决办法是在 .eslintrc.js 文件里面，添加以下代码即可

"env": {
    "browser": true,
    "node": true,
    "commonjs": true,
    "es6": true,        
    "jquery": true  //把jquery添加在这个配置里面
}

注：此配置是eslint的运行环境，可以是浏览器，也可是nodejs，还支持commonjs及es6


b. 'layer' is not defined no-undef

此错误跟第一个错误有点像，但是如果你觉得像jquery那样配置就ok，那你就太天真了。在这里注明一下，layer是贤心写的一个弹窗插件，功能很全。此问题有两种解决方式

/* global layer */    //在报错的文件里面头部写上此注释

或是在.eslintrc.js里面加上此配置

"globals": {
    "layer": true
}


c. 过滤掉不想被检查的文件

那如果我有些库文件不想被检查，应该如何配置？eslint给我们也提供了一个.eslintignore文件，把不想被检查的文件往里面写就可以了。例如，我的是这个样子

/public/js/jquery-3.1.1.js
/public/js/layer

注：默认node_modules文件夹里面的js是不被检查的，所以 node_modules可以不写在此配置里面。


d. 检查多个目录下的文件

如果你想要检查的代码在多个文件夹里，该如何配置？可以在package.json里面，依次写多个目录

"scripts": {
    "start": "supervisor app.js",
    "eslint": "eslint ./*.js config/*.js public/js/*.js routes/*.js"
}

至此，我整个项目里的js文件都可以根据我的配置进行规范了。eslint还有一些高级用法，以后再做记录
